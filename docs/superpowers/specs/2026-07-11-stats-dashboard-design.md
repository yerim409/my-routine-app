# 통계 대시보드 — 설계

날짜: 2026-07-11
상태: 승인됨 (위치: 세 번째 탭, 무드: 클린 베이스 + 게임화 포인트 유지)

## 목표

루틴 수행 기록을 시각화해서 동기부여를 강화한다. 모바일은 `루틴 | 할 일 | 통계`
세 탭, 데스크톱은 헤더의 통계 토글로 2단 레이아웃 ↔ 통계 전체 폭 뷰 전환.

## 범위 (MVP — YAGNI)

1. **잔디 히트맵**: 최근 12주(7×12), 하루에 체크한 루틴 개수 기반 4단계 농도
   (0 = 회색, 1 / 2 / 3+ = emerald 농도). 요일 라벨(월~일), 오늘 테두리 강조.
   과거 시점의 루틴 구성은 알 수 없으므로 달성률(%)이 아닌 **체크 개수**를 쓴다 —
   단순하고 정직한 지표.
2. **주간 요약 카드**: 이번 주 체크 총수 vs 지난주 (증감 화살표),
   이번 주 완벽한 날(모든 매일 루틴 완료) 수.
3. **루틴별 통계 카드**: 루틴마다 현재 스트릭 / **최장 스트릭** / 최근 30일 수행 횟수.
   주 N회 루틴은 주 단위 스트릭(현재/최장)과 최근 4주 달성 주 표시.
4. **투두 한 줄 요약**: 이번 주 완료한 투두 개수. (그 이상은 이번 범위 밖)
   - ⚠️ 현재 `toggleDone`이 `done_at`에 예정일(`when`)을 저장하고 `when` 없으면
     null이라 이 지표가 틀리게 나온다. **`toggleDone`을 실제 완료일
     (`getTodayKey()`) 저장으로 변경**한다 (`TodoTab.jsx` 수정 범위에 포함).
     아카이브 그룹핑은 이미 `when` 기준이라 영향 없음. 기존 데이터의 done_at은
     소급 수정하지 않는다.

제외: 월간 캘린더 뷰, 기간 선택, 루틴별 히트맵, 공유 기능 — 필요해지면 그때.

## 데이터·계산

- StatsTab은 RoutineTab과 동일하게 자체적으로 `routines` + `routine_checks` 로드
  (기존 탭별 독립 로드 패턴 유지). 투두 요약용으로 `todos`의 done/done_at만 추가 로드.
- 계산 헬퍼는 순수 함수로 `src/lib/stats.js` 신설:
  - `buildHeatmap(allChecks, todayKey, weeks=12)` → [{ dateKey, count }] 7×12
  - `calculateLongestStreak(routineId, allChecks, todayKey)` — 일일 루틴 최장 연속
  - `calculateLongestWeeklyStreak(routineId, target, allChecks, todayKey)` — 주 단위 최장
  - `countInRange(routineId, allChecks, fromKey, toKey)` — 기간 수행 횟수
  - 날짜 연산은 기존 `lib/dates.js` 헬퍼 재사용 (로컬 타임존 규칙 동일)
- 최장 스트릭 스캔 범위: 최근 365일 / 52주 (기존 캡과 동일).
- **현재 스트릭 로직 공용화**: RoutineTab 안의 module-private `calculateStreak`
  (일일 현재 스트릭, i===0 유예 규칙 포함)를 `lib/dates.js`로 이동하고
  RoutineTab과 StatsTab이 같은 함수를 import — 로직 중복/드리프트 방지.
- **통계는 항상 `todayKey` 기준** — `selectedDate`(과거 날짜 조회)와 무관.
  통계 뷰가 활성일 때는 헤더의 날짜 선택·"오늘로" 버튼을 숨긴다.
- 주간 증감 비교는 **지난주 같은 요일까지** 누적끼리 비교 (주 초반에 항상
  하락으로 보이는 왜곡 방지).
- 빈 상태: 루틴이 없으면 루틴별 카드 자리에 빈 메시지. "완벽한 날"은
  매일 루틴이 1개 이상인 날만 카운트.
- 히트맵의 미래 셀(이번 주 남은 요일)은 투명 처리, 오늘 셀은 테두리 강조.

## UI

- **모바일 탭**: `🔄 루틴 | ✅ 할 일 | 📊 통계` (기존 탭 바에 추가)
- **데스크톱**: 헤더 우측에 `📊 통계` 토글 버튼. 켜면 main 영역이 통계 뷰(전체 폭,
  내부는 2단 그리드로 카드 배치), 끄면 기존 루틴|할 일 2단으로 복귀
- 상태는 기존 `tab` state 단일 소스 (`'stats'` 값 추가) — 모바일 탭과 데스크톱
  토글이 같은 상태를 공유해 리사이즈 시에도 자연스럽게 이어짐
- 구성 순서: 주간 요약 카드 → 히트맵 카드 → 루틴별 카드 리스트 → 투두 한 줄
- 히트맵 셀은 `aspect-square rounded-[3px]`, 게임화 포인트는 여기 집중
  (완벽한 날 카운트에 ⭐, 증감 화살표 색상)

## Mock 시드 보강

히트맵·스트릭이 보이도록 `supabaseMock.js` 시드에 최근 10주 체크 이력 추가
(결정적 패턴: 물 루틴은 3일 중 2일, 운동은 월/수/금 — 검증 기대값 계산 가능하게).
단, **기존 시드의 기대값은 보존**: 물 루틴은 어제까지 3일 연속으로 끝나야 하고
(현재 스트릭 3), 운동은 지난주 3회 달성 + 이번 주 2회 진행이 유지되어야 한다.
