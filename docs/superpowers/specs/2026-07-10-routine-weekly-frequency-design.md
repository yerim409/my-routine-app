# 루틴 주 N회 반복 규칙 — 설계

날짜: 2026-07-10
상태: 승인됨 (구현 전)

## 목표

지금은 모든 루틴이 암묵적으로 "매일" 루틴이다. 주 3회 운동처럼 매일 하지 않는
루틴을 지원한다. 반복 유형은 **주 N회 (유연)** 하나만 추가한다 — 요일 지정,
N일마다 등은 이번 범위에서 제외 (YAGNI).

## 요구사항

- 루틴은 **매일**(기본) 또는 **주 N회** (N = 1~6) 중 하나
- 주 N회 루틴은 매일 리스트에 표시되고 아무 날이나 체크 가능
- 주 N회 루틴에는 이번 주 진행도(예: `2/3`)를 표시
- 🔥 스트릭: 매일 루틴은 기존대로 연속 일수, 주 N회 루틴은 **연속 달성 주 수**
- 오늘 달성률: 주 N회 루틴은 **체크한 날만** 분자·분모에 포함
- 주 시작 요일은 **월요일**

## 데이터 모델

`routines` 테이블에 컬럼 하나 추가:

```sql
ALTER TABLE routines ADD COLUMN weekly_target SMALLINT;
```

- `NULL` = 매일 루틴 (기존 루틴 전부 자동으로 매일 취급 — 별도 마이그레이션 불필요)
- `1`~`6` = 주 N회 루틴
- `7`은 UI에서 제공하지 않음 (매일과 동일하므로)

`routine_checks`는 변경 없음. 체크 기록 방식(체크한 날짜 row 저장)은 그대로다.

JSONB `repeat_rule` 방식도 검토했으나 기각 — 지금 필요한 규칙이 주 N회 하나뿐이라
과설계이고, 나중에 규칙이 늘면 그때 컬럼을 바꿔도 비용이 크지 않다.

## 동작 설계

### 주(week) 계산

- 주는 월요일 00:00 시작. `getWeekKey(dateKey)` 헬퍼가 해당 날짜가 속한 주의
  월요일 dateKey를 반환한다. 기존 `getDateKey`와 같은 로컬 타임존 기준
  (`new Date(y, m-1, d)` 방식 — UTC 파싱 금지, 기존 타임존 버그 재발 방지).
- 이번 주 수행 횟수 = `allChecks`에서 이번 주 월~일 범위의 날짜 중 해당
  루틴이 체크된 날 수.
- 과거 날짜를 조회 중일 때(`selectedDate`), 진행도 배지와 주간 스트릭은
  **그 dateKey가 속한 주** 기준으로 계산한다 (일일 스트릭이 dateKey 기준인
  것과 동일).

### 스트릭 (주 N회 루틴)

- 🔥 숫자 = 목표(N회)를 채운 주가 몇 주 연속인지. 표시는 `3주`처럼 단위 포함.
- 이번 주가 아직 진행 중이고 목표 미달이어도 스트릭을 깨지 않는다
  (매일 루틴에서 "오늘 아직 안 함"이 스트릭을 안 깨는 것과 같은 원리).
- 🔥 활성(불꽃 컬러) 조건: 마지막 달성 주가 이번 주이거나 지난주일 때.
- 역방향 스캔 상한은 52주 (일일 스트릭의 365일 캡과 동일한 원리).
- 매일 루틴의 스트릭 로직(`calculateStreak`)은 변경 없음.

### 오늘 달성률

- 분모 = 매일 루틴 전체 개수 + **오늘 체크된** 주 N회 루틴 개수
- 분자 = 오늘 체크된 루틴 전체 개수
- 결과: 주 N회 루틴은 체크하면 달성률이 오르고, 안 하면 영향 없음.
  일일 달성률은 "오늘 한 일" 기준을 유지하고, 주간 책임은 주간 스트릭이 담당.

### UI

- **추가 폼**: 이모지·이름 입력 아래에 빈도 선택 추가.
  `매일`(기본 선택) / `주 1회` ~ `주 6회`. 칩(chip) 나열 방식.
- **편집 모드**: 이름/이모지 인라인 편집 시 빈도도 같은 방식으로 수정 가능.
  빈도 변경 시 체크 기록(`routine_checks` row)은 건드리지 않는다. 단,
  `weekly_target`이 단일 컬럼이므로 주간 스트릭은 항상 **현재 N** 기준으로
  과거 주를 재평가한다 (예: 3→5로 올리면 과거 달성 주가 미달로 바뀌어
  스트릭이 소급 감소할 수 있음). 이는 의도된 단순화 — 주별 목표 이력
  저장은 하지 않는다.
- **리스트 아이템**: 주 N회 루틴은 이름 아래에 `이번 주 2/3` 배지.
  목표 달성 시 배지 색을 emerald로 강조.
- **캘린더 모달**: 변경 없음 (주간 달성 하이라이트는 통계 대시보드 작업으로 이연).

## 에러 처리

- `weekly_target`에 1~6 외 값이 저장되지 않도록 UI에서 고정 옵션만 제공.
  기존 optimistic update + Supabase 에러 콘솔 로깅 패턴을 그대로 따른다.
- 스키마 반영 전 앱이 배포될 경우: `weekly_target`이 undefined면 매일 루틴으로
  동작하므로 하위 호환 문제 없음.

## 테스트 / 검증

- 이 프로젝트에는 테스트 러너가 없다. 검증은 dev 서버에서 수동으로:
  1. 매일 루틴 기존 동작(체크/스트릭/달성률) 회귀 확인
  2. 주 3회 루틴 생성 → 진행도 배지 0/3 표시 확인
  3. 3일 체크 → 배지 달성 강조 + 스트릭 반영 확인
  4. 달성률: 주 N회 루틴 체크 전후로 분모 변화 확인
  5. 주 경계(일→월) 넘어갈 때 진행도 리셋 확인 (날짜 선택으로 시뮬레이션)
- 스트릭·주 계산 헬퍼는 순수 함수로 분리해 콘솔에서 검증 가능하게 작성.
