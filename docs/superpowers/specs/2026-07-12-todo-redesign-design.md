# 투두 탭 리디자인

날짜: 2026-07-12
상태: 승인됨 (목업 시안 컨펌 완료)

## 배경

사용자 피드백: 투두 화면이 (1) 생김새가 촌스럽고 (2) 한눈에 안 들어오고 (3) 쓰기 불편함.
Things 3·Todoist 등 유료 앱의 공통 패턴을 기준으로 "돈 내고 쓸 만한" 수준으로 리디자인한다.

## 원인 진단

- 카드마다 그림자+테두리+이모지+파란 날짜+D-day 배지+태그 칩이 전부 보여서 시각적 소음이 큼
- "지난/요일별/다음 주/나중에" 섹션이 모두 같은 비중이라 오늘 할 일이 묻힘
- 추가 폼이 이름+이모지+날짜 2개+태그를 한꺼번에 요구해서 입력 마찰이 큼

## 설계

### 구조 (오늘 중심)

위에서부터:

1. **태그 필터** — 유지하되 작고 절제된 칩으로
2. **오늘 헤더** — "오늘" 큰 타이틀 + 날짜 + "N개 남음"
3. **지난 배너** — `when < today`인 미완료가 있을 때만. 빨간 톤 한 줄
   "지난 할 일 N개" + [오늘로 옮기기](일괄 `when = today`) + 펼치기 토글(개별 확인용)
4. **오늘 섹션** — `when === today`. 히어로 리스트
5. **예정 섹션** — `when > today` 전부, 날짜순 정렬. 요일별 분할 없이 한 리스트로,
   행 오른쪽에 날짜 라벨(내일 / 수 7/15)
6. **나중에 섹션** — `when == null`. 기본 접힘(개수만 표시), 탭하면 펼침
7. **퀵 애드 바**
8. **완료 아카이브** — 기존 TodoArchive 그대로

전체 리스트는 흰 패널 컨테이너 하나에 담는다 (민트 배경 위 카드 난립 제거).

### 행 디자인

- 카드 → 구분선(divider) 리스트 행. 그림자 없음
- 왼쪽: 드래그 핸들(연한 회색, hover 시 진해짐) + 체크 서클(22px, 체크 시 에메랄드 채움 + pop 애니메이션)
- 가운데: 할 일 이름 한 줄
- 오른쪽 메타: D-day 필(기한 있을 때만, 임박 시 빨강/아니면 회색) + 태그 색 점(7px)
- 이모지는 표시·입력 모두 제거 (DB 데이터는 보존, UI에서만 미사용)
- 삭제 버튼은 행에서 제거 → 행 클릭 시 열리는 인라인 편집 폼 안으로 이동
- 요일별 색색 섹션 라벨 제거. 색은 D-day 경고(빨강)와 태그 점에만 사용

### 퀵 애드

- 한 줄 인풋 상시 노출: 이름 입력 후 Enter → 즉시 추가, 포커스 유지(연속 추가)
- 인풋 아래 칩 행: 날짜 선택 [오늘|내일|날짜(native picker)|나중에] (기본 오늘, 상호배타)
  + [기한] 토글(누르면 date input 노출) + 태그 칩 토글 + 새 태그
- 추가 후에도 칩 선택 상태 유지 (같은 날짜/태그로 연속 입력)

### 편집

기존 인라인 편집 유지하되: 이모지 필드 제거, 삭제 버튼 추가, 스타일 통일.
